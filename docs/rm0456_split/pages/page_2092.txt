RM0456

Bit 0 INITOK: PKA initialization OK
This bit is asserted when PKA initialization is complete.
0: PKA is not initialized correctly. START bit cannot be set.
1: PKA is initialized correctly and can be used normally.
Note: When RNG is not able to output proper random numbers or when SAES is using the RNG bus,
INITOK remains at 0. If it happens, make sure the RNG peripheral is properly clocked and
initialized, and the SAES peripheral is clocked with BUSY bit cleared in SAES_SR to release
the RNG bus, then enable PKA again.

Bit 20 ADDRERRFC: Clear address error flag

53.7.4

PKA RAM
The PKA RAM is mapped at the offset address of 0x0400 compared to the PKA base
address. Only 32-bit word single accesses are supported, through PKA.AHB interface.
RAM size is 5336 bytes (max word offset: 0x14D0)
