3.8.1

Hardware secret key management
As shown in the previous figure, the devices propose a better protection for application
keys, using hardware secret keys. These AES keys can be made usable to the application,
without exposing them in clear-text (unencrypted). Such keys also become immediately
unusable in case of tamper.
There are three different sources of hardware secret keys:
•

DHUK: derived keys based on 256-bit nonvolatile device unique secret in flash
memory. The generation of this key takes into account the TrustZone state and key use
state (KMOD).
Note: DHUK is the same for all devices when RDP = 0 (debug/development mode).

•

BHK: 256-bit application key stored in tamper-resistant volatile storage in TAMP
This key is written at boot time, then read/write locked to application until next reset.

•

XORK: result of an XOR of BHK and DHUK

These keys can be used:
•
